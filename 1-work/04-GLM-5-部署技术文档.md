## 5. 部署方案

### 5.1 方案一：vLLM 部署（推荐）

#### 5.1.1 单机多卡部署

```bash
# 基础启动命令（8卡A100）
python -m vllm.entrypoints.openai.api_server \
    --model /data/models/GLM-5-Instruct \
    --tensor-parallel-size 8 \
    --trust-remote-code \
    --port 8000 \
    --host 0.0.0.0 \
    --max-model-len 32768 \
    --gpu-memory-utilization 0.95

# 后台运行
nohup python -m vllm.entrypoints.openai.api_server \
    --model /data/models/GLM-5-Instruct \
    --tensor-parallel-size 8 \
    --trust-remote-code \
    --port 8000 \
    --host 0.0.0.0 \
    --max-model-len 65536 \
    > vllm_glm5.log 2>&1 &
```

#### 5.1.2 关键参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `--tensor-parallel-size` | 张量并行卡数 | 根据GPU数量 |
| `--max-model-len` | 最大上下文长度 | 32768/65536 |
| `--gpu-memory-utilization` | GPU显存利用率 | 0.90-0.95 |
| `--max-num-seqs` | 最大并发序列数 | 256 |
| `--dtype` | 数据类型 | auto/bfloat16 |

#### 5.1.3 多节点部署

```bash
# 节点1（主节点）
python -m vllm.entrypoints.openai.api_server \
    --model /data/models/GLM-5-Instruct \
    --tensor-parallel-size 16 \
    --pipeline-parallel-size 2 \
    --trust-remote-code \
    --port 8000

# 节点2（从节点）- 自动发现，无需额外配置
```

#### 5.2.1 单容器部署

```bash

    --tensor-parallel-size 8 \
    --trust-remote-code \
    --host 0.0.0.0 \
    --port 8000 \
    --max-model-len 32768
```

```bash
# Chat Completion

  -H "Content-Type: application/json" \
  -d '{
```

### 6.2 Python SDK 调用

```python
from openai import OpenAI

# 初始化客户端
client = OpenAI(
    api_key="your-api-key",  # 可自定义
    base_url="http://localhost:8000/v1"
)

# 同步调用
response = client.chat.completions.create(
    model="GLM-5-Instruct",
    messages=[
        {"role": "system", "content": "你是一个有帮助的AI助手。"},
        {"role": "user", "content": "请解释什么是量子计算"}
    ],
    temperature=0.7,
    max_tokens=2048
)
print(response.choices[0].message.content)

# 流式调用
stream = client.chat.completions.create(
    model="GLM-5-Instruct",
    messages=[
        {"role": "user", "content": "写一首关于AI的诗"}
    ],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

## 7. 性能优化

### 7.1 量化部署

```bash

    --model zai-org/GLM-5-FP8 \
    --tensor-parallel-size 4 \

    --trust-remote-code
```

```bash
# 使用 KV Cache 优化
python -m vllm.entrypoints.openai.api_server \
    --model /data/models/GLM-5-Instruct \
    --tensor-parallel-size 8 \
    --enable-prefix-caching \
    --gpu-memory-utilization 0.90 \
    --max-model-len 65536 \
```

## 8. 常见问题

### 8.1 显存不足 (OOM)

```bash
# 问题现象
# RuntimeError: CUDA out of memory

# 解决方案
# 1. 降低 max-model-len
--max-model-len 16384

# 2. 降低 gpu-memory-utilization
--gpu-memory-utilization 0.85

# 3. 使用量化模型
--quantization awq

# 4. 启用 offloading
--enable-offload
```

### 8.2 多卡通信问题

```bash
# 问题现象
# NCCL error / GPU P2P通信失败

# 解决方案
# 1. 检查 NCCL 配置
export NCCL_DEBUG=INFO
export NCCL_P2P_DISABLE=0
export NCCL_IB_DISABLE=0

# 2. 使用 NVLink
export NCCL_P2P_LEVEL=NVL

# 3. 禁用 P2P（降级方案）
export NCCL_P2P_DISABLE=1
```

### 8.3 模型加载失败

```bash
# 问题现象
# ValueError: trust_remote_code is disabled

# 解决方案
# 添加 trust-remote-code 参数
--trust-remote-code

# 或设置环境变量
export TRANSFORMERS_TRUST_REMOTE_CODE=true
```

### 8.4 性能不达预期

```bash
# 排查步骤
# 1. 检查 GPU 利用率
nvidia-smi -l 1

# 2. 检查显存带宽
nvidia-smi dmon -s u

# 3. 启用性能分析
export VLLM_PROFILER_ENABLED=true

# 4. 调整批处理参数
--max-num-seqs 256
--max-num-batched-tokens 65536
```

### 8.5 健康检查脚本

```bash
#!/bin/bash
# health_check.sh

ENDPOINT="http://localhost:8000"
MAX_RETRIES=30
RETRY_INTERVAL=10

check_health() {
    curl -s -o /dev/null -w "%{http_code}" "$ENDPOINT/health"
}

echo "等待 GLM-5 服务启动..."

for i in $(seq 1 $MAX_RETRIES); do
    STATUS=$(check_health)
    if [ "$STATUS" == "200" ]; then
        echo "✅ 服务已就绪！"

        # 测试 API
        echo "测试 API 调用..."
        curl -s "$ENDPOINT/v1/chat/completions" \
            -H "Content-Type: application/json" \
            -d '{"model":"GLM-5-Instruct","messages":[{"role":"user","content":"hi"}],"max_tokens":10}' \
            | jq .

        exit 0
    fi
    echo "等待中... ($i/$MAX_RETRIES) 状态码: $STATUS"
    sleep $RETRY_INTERVAL
done

echo "❌ 服务启动超时"
exit 1
```
